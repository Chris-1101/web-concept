-el diseño

Por lo demás, diseñe el sitio desde cero. Elegí usar colores oscuros y temática 'Metro' (un lenguaje de diseño de Microsoft, popular por su uso en Windows 8). Metro Design Language.

-tipografías

Las fuentes usadas en el diseño son: Apex New y Segoe UI. Estas fuentes no son gratuitas, pero tengo licencia que me permiten usarlas en el diseño, pero no para incrustarlas en el sitio web. Por eso tuve que usar fuentes diferentes para el texto general.

Para el texto del sitio, incruste dos 'webfonts' muy populares en la red ahora mismo, que son Open Sans y Lato. Quería tener un sitio independiente, así que me las subí junto al sitio en vez de tener un lienzo hacia Google.

-scripts

También use algunas librerías de JavaScript, que por supuesto no las escribí yo. SWF Object para incrustar objetos Flash de manera más limpia, y JQuery 1.9.1 para el JQuery. También está el script de mi cuenta en Google Analytics.

HTML 5

-estructura

Además del logo Flash y del banner de anuncio, La cabecera lleva unos elementos animados con CSS para el inicio de la pagina, y los iconos / lienzos de redes sociales.

El contenido lleva tres páginas (un portfolio de clientes recientes, una página donde se detallan los servicios ofrecidos, y un formulario de contacto) que están definidas todas en el mismo documento HTML.

-cabecera y metas

Use todos los meta-tags que usaría normalmente en un sitio de verdad. Están los habituales, pero también está la meta de verificación de mi cuenta en Google Webmaster Tools y la de viewport que es un ajuste para el 'Responsive Design' y móviles.

-etiquetaje

La estructura del sitio lleva dos partes.

La parte de arriba, incluyendo el visor, es el <header>, que lleva el logo hecho con Flash, el banner de anuncio Flash , y la parte del visor <div id="viewport">. El visor a su vez, contiene los elementos animados de css.

La parte debajo del visor es un <section>, que contiene el contenido principal del sitio, con el <nav> al principio, y el <footer> al final.

CSS 3

Dividí el CSS en tres documentos basado en el tipo de reglas que se estaba definiendo.

-general y 'responsive'

El documento principal de CSS default.css que contiene todas las reglas básicas para la estructura y el estilo del sitio. Al final están definidas las reglas @media que manejan diferentes estilos basadas en la anchura de la ventana del navegador ('Responsive Design').

-animaciones

Un documento CSS aparte animations.css que contiene todas las reglas para las animaciones de inicio de la cabecera. Decidí ponerlos aparte porque es mucho código para poca cosa (con los prefijos de cada navegador de la regla @keyframes) y para poder quitarlos fácilmente (que me vino bien durante el desarrollo).

-tipografías

Y un último documento CSS fonts.css para las tipografías incrustadas mediante la regla @font-face. Igual que con las animaciones, siempre me parece mucho código por poca cosa, y ya que se define una vez y se olvida de ello, lo pongo aparte.

Flash / Scripts

Escribí algunos script para facilitar la interacción del usuario ya que sabía como, pero son todas cosas muy sencillas de unas cuantas líneas de código. Ya sé que no toca en el curso y que no se va a valorar, pero lo explico rápido.

-objetos flash (javascript)

Incrustar objetos Flash suele llevar 20 líneas de código para muy poca cosa, y como me gusta tener el código lo más limpio y simple posible, use una librería de Google que te permite hacerlo llamando a una función sin más. El script chris.flash.js mas que código es una hoja de ajustes, pero lo comento en el archivo mismo si quieres ver cómo funciona.

-scroll lento (jquery)

El lienzo en la cabecera 'mis trabajos' hace que la pagina haga un scroll hacia el menú de navegación del contenido. Pero lo anima lentamente, en vez de saltar directamente a ese lugar como pasaría usando simples <a name="ancla">. Eso lo consigo con el método scrollTop() de JQuery. Son 2 líneas por cada lienzo (el de inicio para volver a subir usa el mismo), lo comento en el archivo chris.scroll.js.

-pestañas (jquery)

Escribí un script usando JQuery ( chris.tabs.js ), para hacer transiciones entre páginas sin tener que cargar una página nueva. Un poco como tener pestañas para navegar entre diferentes secciones. Lo comento en el archivo mismo.

-email (php)

El script de php que escribí para el formulario ( chris.email.php ) se parece mucho al que nos distes tu. La única diferencia es que usé el método predefinido str.replace() para reemplazar a los saltos de línea del textarea (que no serán reconocidos por HTML) por <br> y la función predefinida header() para redirigir al usuario a una página de confirmación de envío. Lo comento en el archivo mismo.
